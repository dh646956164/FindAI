# FindAI 项目启动说明

项目目录：

```text
C:\Users\Dinghao\Documents\FindAI
```

日常开发地址：<http://127.0.0.1:3001>

## 一、电脑重启后如何启动

### 1. 启动数据库

如果项目使用 Docker Desktop 中的 PostgreSQL：

1. 打开 Docker Desktop。
2. 等待 Docker Desktop 完全启动。
3. 打开 PowerShell，执行：

```powershell
cd "C:\Users\Dinghao\Documents\FindAI"
docker compose up -d postgres
```

如果使用的是单独安装的 PostgreSQL 或云数据库，只需确认数据库服务可以连接，不需要执行 `docker compose`。

### 2. 启动网站

在 PowerShell 中执行：

```powershell
cd "C:\Users\Dinghao\Documents\FindAI"
npm run dev -- -p 3001
```

终端出现 `Ready` 后，打开浏览器访问：

```text
http://127.0.0.1:3001
```

启动网站的 PowerShell 窗口需要保持打开。

## 二、如何停止网站

在运行网站的 PowerShell 窗口中按：

```text
Ctrl + C
```

如果出现确认提示，输入 `Y` 并回车。

如需同时停止 Docker 数据库，可执行：

```powershell
docker compose stop postgres
```

停止数据库不会删除已有数据。

## 三、首次运行或依赖丢失

只有首次下载项目、删除了 `node_modules`，或者依赖发生变化时，才需要执行：

```powershell
cd "C:\Users\Dinghao\Documents\FindAI"
npm install
```

然后再启动网站：

```powershell
npm run dev -- -p 3001
```

## 四、首次初始化数据库

数据库只需要初始化一次，不要在每次启动时重复执行。

确认 `.env` 已正确配置后执行：

```powershell
cd "C:\Users\Dinghao\Documents\FindAI"
npm run db:init
```

如果只是更新了 seed 示例数据，可以执行：

```powershell
npm run db:seed
```

## 五、页面没有样式或校徽异常变大

这通常是开发服务器缓存或 CSS 文件没有加载。处理步骤：

1. 在 PowerShell 中按 `Ctrl + C` 停止项目。
2. 重新执行：

```powershell
cd "C:\Users\Dinghao\Documents\FindAI"
npm run dev -- -p 3001
```

3. 浏览器中按 `Ctrl + F5` 强制刷新。

不要在 `npm run dev` 正在运行时同时执行 `npm run build`，否则可能覆盖开发服务器正在使用的 `.next` 文件。

## 六、端口 3001 被占用

查看占用端口的进程：

```powershell
Get-NetTCPConnection -LocalPort 3001 -State Listen
```

也可以临时换用 3002 端口：

```powershell
npm run dev -- -p 3002
```

然后访问：<http://127.0.0.1:3002>

## 七、生产构建检查

需要检查项目能否正式构建时，先停止开发服务器，再执行：

```powershell
cd "C:\Users\Dinghao\Documents\FindAI"
npm run typecheck
npm run build
```

构建完成后，如需继续开发，再重新执行：

```powershell
npm run dev -- -p 3001
```
